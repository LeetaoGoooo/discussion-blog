快捷指令 + Dify +飞书多维表格 = 稍后阅读

By LeetaoGoooo on February 16, 2025

前言

之前折腾好多个稍后阅读软件：Pocket、Notion、instapaper、Matter 以及 Omnivore，后来 Omnivore 停服，又一度迁移回到了 Pocket。最近恰好赶上 Deepseek 热潮，看的不少文章介绍飞书多维表格接入 Deepseek 的新闻，蠢蠢欲动了起来，就开始折腾起来了飞书多维表格，在折腾过程中突然意识到我可以将稍后阅读功能放到飞书多维表格完成（在我不需要离线的前提下），并且借助 AI 我还可以完成自动打标签以及总结的功能。

稍后阅读

v1 飞书多维表格

最初考虑的功能最简单，配合飞书表格自带的功能，实现的逻辑就是当添加一个链接的时候，会爬取文章的对应的内容，然后借助 AI 去实现大纲的总结和自动标签。

后面我需要做的就是遇到好的文章然后打开飞书添加链接就好了。

v2 机器人 + 飞书表格

但是这远远还不够，因为需要好几个步骤，我希望是可以一步完成这个步骤，所以最好有个 webhook。好消息是 webhook 它真的有，坏消息就是要付费。

付费是可能的，不过是最后的方案。随后就开始调研，有没有多维表格的 Api，发现是有的，所以就考虑使用做一个机器人去完成这些，但是我又懒得自建服务器，我又找有没有更简洁的方案，接着我发现了飞书机器人这个应用

这个机器人有 webhook，每个月有 200 次的免费额度，有新增多维表格记录的操作，看起来很不错（实际上有坑），很快我就搭建了第一个机器人。

但是问题来了，这里的 webhook 支持的参数类型只是 json 的基本类型，但是我之前的使用的多维表格的文章链接是多维表格自带的 URL 类型，是无法通过 webhook 新增的，并且也不存在任何字段或者公式可以将 string 类型的链接转换成 URL 类型。

所以飞书机器人 + 多维表格 这个方案行不通。

v3 快捷指令 + Dify + 机器人 + 飞书表格

如果有个接口可以把对应的链接内容提取，然后完成总结，再回调机器人就完美了，然后我就想起来很久之前在群里看到的 Dify。我就开始了研究起来了 Dify。Dify 支持通过 Api 直接调用，也可以完成解析 url 内容，调用 LLM 完成总结和自动打标签的功能。
